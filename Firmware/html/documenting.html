C Code Documentation with Doxygen

Documenting SW and FW is done directly in the code by using special tags that can be extracted by a tool, which copiles a html documentation.
This guarantees that the documentation is up to date.
Such a tool is Doxygen http://www.doxygen.nl
Introduction: http://www.doxygen.nl/manual/docblocks.html
Reference: http://www.doxygen.nl/manual/commands.html
Note: The STM32 microcontroller libraries and this demo_code have been documented with Doxygen.

Install and Run Doxygen

Download and install http://www.doxygen.nl
Doxygen does not have a fixed associated file name extension. To open a configuration file: Start the program Doxywizard
A suitable Doxygen configuration file named Doxyfile is already in the demo_code folder. You may open it from within Doxywizard and adapt it to your needs. (File => Open or afterwards File => Open recent)
If needed edit the settings in the registers Wizard and/or Expert. Save the changes: File => Save
Register Run => Run doxygen compiles the documentation
Register Run => Show HTML output opens the documentation in a browser

Dependency Graphs

How to use Doxygen

The content (purpose, task) of every C- and H-file should be documented on the top of the file.


Each function should be preceeded by ist function header. It should be made clear what a function does, which parameters it expects and what result it returns.


Constants and variables can be documented directly inline after ///<

Related Pages

The folder demo_code/html/doc contains some files that you have to edit to reflect your prototype.
main.dox is used as the main page in the documentation and should give a good overview of the project.
testing.dox is the documentation of the tests that you have done.
You can add other files if required, such as a FAQ or a user manual. They are automatically added to the Related Pages tab by Doxygen.
